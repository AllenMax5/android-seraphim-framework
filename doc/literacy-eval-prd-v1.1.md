## 1. 产品概述

### 1.1 产品定位

一款面向小学教育的**跨学科素养评价数字化工具**，基于"四维·三阶·六翼"
评价模型，为教师、学生、家长及学校管理者提供全方位的素养评价解决方案。

**核心价值主张**：

- 📊 **科学性**：基于权威评价模型的专业评价体系
- ⚡ **便捷性**：支持扫码即用，离线缓存能力
- 📈 **可视化**：雷达图、成长树等直观展示发展轨迹
- ⚙️ **个性化**：六翼维度可自定义，适配不同学校

### 1.2 目标用户画像

#### 1.2.1 教师用户

**基本信息**：

- 年龄：25-45岁
- 职业：小学教师、班主任、学科组长
- 技术能力：中等，熟练使用智能手机和基础办公软件
- 痛点：评价记录繁琐、数据分析困难、家校沟通低效

**典型场景**：
> 王老师，35岁，三年级班主任。每天需要记录30+学生的课堂表现，期末需要手动统计每个学生的素质报告，耗时耗力。

**核心需求**：

1. 快速记录学生表现（30秒内完成单次评价）
2. 自动生成班级数据报告
3. 便捷的家校沟通工具
4. 离线使用能力（教室网络不稳定）

#### 1.2.2 家长用户

**基本信息**：

- 年龄：28-42岁
- 职业：各行业从业者
- 技术能力：中等偏上，熟练使用微信等社交软件
- 痛点：不了解孩子在校表现、难以配合学校教育

**典型场景**：
> 李妈妈，36岁，企业职员。想了解孩子在学校的表现，但只能通过家长会或偶尔的老师沟通，信息滞后且不全面。

**核心需求**：

1. 实时了解孩子在校表现
2. 获取个性化的家庭教育建议
3. 记录孩子的校外成长
4. 与教师便捷沟通

#### 1.2.3 学生用户

**基本信息**：

- 年龄：6-12岁
- 年级：小学1-6年级
- 技术能力：基础，能使用简单的教育类App
- 特点：喜欢游戏化、视觉化、即时反馈

**典型场景**：
> 小明，9岁，三年级学生。喜欢看到自己的进步被认可，希望能像玩游戏一样"升级成长"。

**核心需求**：

1. 看到自己的努力被记录和认可
2. 游戏化的成长激励
3. 展示自己的作品
4. 获得成就徽章

#### 1.2.4 管理员用户

**基本信息**：

- 年龄：35-50岁
- 职业：教务主任、校长、教研员
- 技术能力：中等，熟悉教育管理系统
- 痛点：缺乏全面的数据支撑决策、评价体系难以落地

**核心需求**：

1. 全校数据一目了然
2. 评价体系的灵活配置
3. 支持教育科研的数据导出
4. 教师使用情况监控

## 2. 市场分析

### 2.1 市场背景

**政策驱动**：

- 《深化新时代教育评价改革总体方案》（2020年）明确提出"改进结果评价，强化过程评价"
- "双减"政策后，素质教育评价需求激增
- 各地教育局推动学生综合素质评价数字化

**市场规模**：

- 中国小学约15万所，小学生约1亿人
- 教育信息化市场规模超5000亿元，年增长率15%+
- 素质教育评价细分市场预估超100亿元

### 2.2 竞品分析

| 竞品        | 定位   | 优势         | 劣势            | 我们的差异化    |
|-----------|------|------------|---------------|-----------|
| 某综合素质评价平台 | 教育局级 | 功能全面、数据权威  | 使用复杂、体验差、无游戏化 | 体验好、游戏化激励 |
| 某家校沟通App  | 家校沟通 | 用户基数大、功能轻量 | 评价维度单一、无专业模型  | 专业评价体系    |
| 某成长档案平台   | 成长记录 | 界面美观、操作便捷  | 评价模型简单、无数据分析  | 四维三阶六翼模型  |

### 2.3 SWOT 分析

**优势 (Strengths)**：

- 独创的"四维·三阶·六翼"评价模型
- 游戏化激励机制
- 离线缓存能力
- 可配置化适配不同学校

**劣势 (Weaknesses)**：

- 新品牌知名度低
- 初期学校资源有限
- 技术团队规模较小

**机会 (Opportunities)**：

- 政策红利期
- 素质教育评价需求增长
- 学校数字化转型趋势

**威胁 (Threats)**：

- 大厂入局（钉钉、企业微信等）
- 政策变化风险
- 数据安全合规要求提高

## 3. 核心模型

### 3.1 四维·三阶·六翼 评价模型

```
┌─────────────────────────────────────────────────────────────────┐
│                        评价模型架构                               │
├─────────────────────────────────────────────────────────────────┤
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  六翼（一级指标）                                          │  │
│  │  学科基础 │ 艺术审美 │ 科技创新 │ 身心健康 │ 社会实践 │ 人文底蕴 │  │
│  └───────────────────────────────────────────────────────────┘  │
│                              ↓                                   │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  四维（综合能力）                                          │  │
│  │     学习能力     │     实践能力     │     创新能力     │     合作能力     │  │
│  └───────────────────────────────────────────────────────────┘  │
│                              ↓                                   │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  三阶（进阶等级）                                          │  │
│  │     基础达成     →     良好发展     →     卓越表现           │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 六翼（一级指标）

六翼代表六个核心素养维度，可根据学校特色进行配置：

| 维度       | 说明            | 示例       | 权重范围   | 评价要点           |
|----------|---------------|----------|--------|----------------|
| **学科基础** | 语文、数学、英语等学科素养 | 语数英核心能力  | 15-20% | 课堂表现、作业质量、测验成绩 |
| **艺术审美** | 音乐、美术、舞蹈等     | 音美舞艺术修养  | 10-15% | 作品创作、表演展示、审美表达 |
| **科技创新** | 科学、编程、创客等     | 科学编程创客能力 | 15-20% | 探究实验、创意制作、问题解决 |
| **身心健康** | 体育、心理健康等      | 体健心健康素养  | 15-20% | 体能测试、运动参与、心理状态 |
| **社会实践** | 劳动教育、志愿服务等    | 劳育志愿服务   | 15-20% | 劳动表现、社区服务、团队合作 |
| **人文底蕴** | 阅读、传统文化等      | 阅读传统文化   | 10-15% | 阅读量、经典诵读、文化理解  |

**配置化特性**：

- 学校可在后台修改六翼名称和权重
- 支持不同年级差异化配置
- 可启用/禁用特定维度

### 3.3 四维（综合能力）

将六翼具体表现映射到四个综合能力维度：

**算法公式**：

```
学习能力 = 学科基础×0.4 + 人文底蕴×0.3 + 科技创新×0.2 + 艺术审美×0.1
实践能力 = 社会实践×0.4 + 身心健康×0.3 + 科技创新×0.2 + 学科基础×0.1
创新能力 = 科技创新×0.4 + 艺术审美×0.3 + 学科基础×0.2 + 人文底蕴×0.1
合作能力 = 社会实践×0.4 + 身心健康×0.3 + 艺术审美×0.2 + 学科基础×0.1
```

**四维雷达图展示**：

```
                学习能力
                   ▲
                  /|\
                 / | \
                /  |  \
               /   |   \
    合作能力 ◄────┼────► 创新能力
               \   |   /
                \  |  /
                 \ | /
                  \|/
                   ▼
                实践能力
```

### 3.4 三阶（进阶等级）

| 等级          | 名称   | 分数区间    | 能力描述          | 视觉标识  |
|-------------|------|---------|---------------|-------|
| **Level 1** | 基础达成 | 60-75分  | 掌握基本要求，具备基础能力 | 🥉 铜星 |
| **Level 2** | 良好发展 | 76-89分  | 表现良好，能力稳步发展   | 🥈 银星 |
| **Level 3** | 卓越表现 | 90-100分 | 表现优异，具备突出能力   | 🥇 金星 |

### 3.5 四级场景（评价场景）

| 层级     | 场景类型 | 说明                 | 评价频次 | 数据占比 | 评价方式   |
|--------|------|--------------------|------|------|--------|
| **L1** | 基础层  | 日常课堂表现、作业完成情况      | 每日   | 40%  | 教师实时评价 |
| **L2** | 活动层  | 艺术节、益智节、阅读节、科技节等   | 每月   | 30%  | 活动专项评价 |
| **L3** | 项目层  | 寒暑假"1+1"任务、STEAM项目 | 每学期  | 20%  | 作品评审   |
| **L4** | 实践层  | 校-家-社-村实践活动        | 每季度  | 10%  | 打卡+审核  |

### 4.3 教师端功能详细设计

#### 4.3.1 扫码/选人评价（P0）

**功能编号**：T-F001  
**功能名称**：快速评价  
**功能描述**：在活动场景中快速评价学生

**用户故事**：

> **US-001**：作为班主任，我希望在艺术节现场通过扫码快速评价学生，以便及时记录学生的表现，而不是事后回忆记录。

> **US-002**：作为任课教师，我希望在课堂中快速记录学生的表现，每次评价不超过30秒，不影响教学进度。

> **US-003**：作为教师，我希望在网络不稳定的情况下也能记录评价，联网后自动同步，确保数据不丢失。

**业务流程**：

```

  ▼

  ▼

  │
  ▼

  ▼

```

**界面原型**：[见 5.1 教师端原型]

**功能验收标准**：

| 验收项  | 验收标准            | 优先级 |
|------|-----------------|-----|

**功能编号**：T-F002  
**功能名称**：批量评价  
**功能描述**：全班测试、课堂表现等场景批量录入

**用户故事**：

> **US-004**：作为体育老师，我希望一次性录入全班学生的体能测试成绩，避免重复操作，提高效率。

> **US-005**：作为教师，我希望导入Excel格式的成绩表，系统自动批量录入评价。

**核心流程**：

```
[选择评价活动]
  │
  ▼
[批量选择学生] ──▶ (全班/自定义选择/按小组)
  │
  ▼
```

**性能要求**：

- 支持单次批量评价 ≤ 50人

## 11. 运营策略

### 11.1 推广计划

**第一阶段：试点学校（1-3月）**

- 选择3-5所合作学校免费试用
- 收集反馈，快速迭代
- 打造标杆案例

**第二阶段：区域推广（4-6月）**

- 举办区域教育信息化会议演示
- 与教育局合作推广
- 学校间口碑传播

**第三阶段：全国拓展（7-12月）**

- 线上营销推广
- 渠道合作伙伴
- 行业会议展示

### 11.2 商业模式

**定价策略**：

- 基础版：免费（单校≤100学生）
- 标准版：5元/学生/学期
- 高级版：10元/学生/学期（含数据分析）
- 定制版：按需报价

**收入来源**：

- 学校订阅费（主要）
- 增值服务（培训、定制开发）
- 数据服务（区域教育报告）
