## 1. 产品概述

### 1.1 产品定位

一款面向小学教育的**跨学科素养评价数字化工具**，基于"四维·三阶·六翼"
评价模型，为教师、学生、家长及学校管理者提供全方位的素养评价解决方案。

**核心价值主张**：

- 科学性：基于权威评价模型的专业评价体系
- 便捷性：支持扫码即用，离线缓存能力
- 可视化：雷达图、成长树等直观展示发展轨迹
- 个性化：六翼维度可自定义，适配不同学校

### 1.2 目标用户

| 用户角色  | 核心需求           | 使用场景       | 使用频率 |
|-------|----------------|------------|------|
| 教师    | 快速评价、批量录入、数据分析 | 课堂、活动、日常观察 | 每日   |
| 学生    | 成长记录、作品展示、获得激励 | 课堂表现、活动参与  | 每日   |
| 家长    | 了解孩子发展、配合家校共育  | 查看报告、参与活动  | 每周   |
| 学校管理员 | 体系配置、数据管理、科研分析 | 后台管理、数据导出  | 每月   |

### 1.3 核心价值矩阵

| 价值维度 | 教师端   | 学生端   | 家长端   | 管理端   |
|------|-------|-------|-------|-------|

## 4. 核心模型

### 4.1 四维·三阶·六翼 评价模型

```
┌─────────────────────────────────────────────────────────────────┐
│                        评价模型架构                               │
├─────────────────────────────────────────────────────────────────┤
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  六翼（一级指标）                                          │  │
│  │  学科基础 │ 艺术审美 │ 科技创新 │ 身心健康 │ 社会实践 │ 人文底蕴 │  │
│  └───────────────────────────────────────────────────────────┘  │
│                              ↓                                   │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  四维（综合能力）                                          │  │
│  │     学习能力     │     实践能力     │     创新能力     │     合作能力     │  │
│  └───────────────────────────────────────────────────────────┘  │
│                              ↓                                   │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  三阶（进阶等级）                                          │  │
│  │     基础达成     →     良好发展     →     卓越表现           │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

#### 4.1.1 六翼（一级指标）

六翼代表六个核心素养维度，可根据学校特色进行配置：

| 维度   | 说明            | 示例       | 权重范围   |
|------|---------------|----------|--------|
| 学科基础 | 语文、数学、英语等学科素养 | 语数英核心能力  | 15-20% |
| 艺术审美 | 音乐、美术、舞蹈等     | 音美舞艺术修养  | 10-15% |
| 科技创新 | 科学、编程、创客等     | 科学编程创客能力 | 15-20% |
| 身心健康 | 体育、心理健康等      | 体健心健康素养  | 15-20% |
| 社会实践 | 劳动教育、志愿服务等    | 劳育志愿服务   | 15-20% |
| 人文底蕴 | 阅读、传统文化等      | 阅读传统文化   | 10-15% |

**配置化特性**：学校可在后台修改六翼名称和权重，适配本校育人目标。

#### 4.1.2 四级场景（评价场景）

| 层级 | 场景类型 | 说明                 | 评价频次 | 数据占比 |
|----|------|--------------------|------|------|
| L1 | 基础层  | 日常课堂表现、作业完成情况      | 每日   | 40%  |
| L2 | 活动层  | 艺术节、益智节、阅读节、科技节等   | 每月   | 30%  |
| L3 | 项目层  | 寒暑假"1+1"任务、STEAM项目 | 每学期  | 20%  |
| L4 | 实践层  | 校-家-社-村实践活动        | 每季度  | 10%  |

#### 4.1.3 三阶进阶标准

| 等级      | 名称   | 分数区间    | 能力描述          |
|---------|------|---------|---------------|
| Level 1 | 基础达成 | 60-75分  | 掌握基本要求，具备基础能力 |
| Level 2 | 良好发展 | 76-89分  | 表现良好，能力稳步发展   |
| Level 3 | 卓越表现 | 90-100分 | 表现优异，具备突出能力   |

#### 4.1.4 四维综合得分算法

将六翼具体表现映射到四个综合能力维度：

```
学习能力 = 学科基础×0.4 + 人文底蕴×0.3 + 科技创新×0.2 + 艺术审美×0.1
实践能力 = 社会实践×0.4 + 身心健康×0.3 + 科技创新×0.2 + 学科基础×0.1
创新能力 = 科技创新×0.4 + 艺术审美×0.3 + 学科基础×0.2 + 人文底蕴×0.1
合作能力 = 社会实践×0.4 + 身心健康×0.3 + 艺术审美×0.2 + 学科基础×0.1
```

### 5.2 教师端功能

#### 5.2.1 扫码/选人评价（P0）

**功能描述**：在活动场景中快速评价学生

**用户故事**：
> 作为班主任，我希望在艺术节现场快速对学生进行素养评价，以便及时记录学生的表现。

**核心流程**：

1. 选择评价场景（如"艺术节评价"、"阅读节评价"）
2. 选择班级
3. 选择学生（扫码或列表选择）
4. 选择对应六翼指标
5. 一键打分（支持三级进阶）
6. 添加评语（可选）
7. 提交评价

**界面元素**：

- 扫码入口（支持学生二维码扫描）
- 班级/学生选择器
- 六翼指标卡片（彩色区分）
- 三级进阶评分（⭐/⭐⭐/⭐⭐⭐）
- 快捷评语标签
- 语音评价输入

**验收标准**：

- [ ] 单次评价完成时间 ≤ 30秒
- [ ] 支持离线评价，联网后自动同步
- [ ] 评价提交后学生端实时收到通知

#### 5.2.2 批量评价（P1）

**功能描述**：全班跳绳测试、课堂表现等场景批量录入

**用户故事**：
> 作为体育老师，我希望一次性录入全班学生的跳绳成绩，避免重复操作。

**核心流程**：

1. 选择评价活动
2. 批量选择学生（全班/多选）
3. 统一或分别评分
4. 批量提交

**界面元素**：

- 学生列表（支持多选/全选）
- 批量评分面板
- 快速录入模式
- Excel导入模板

#### 5.2.3 活动任务发布（P1）

**功能描述**：发布寒暑假"1+1"任务、STEAM项目等

**用户故事**：
> 作为学科组长，我希望发布假期实践任务，让学生上传作品后进行线上评价。

**核心流程**：

1. 创建任务（标题、描述、要求、截止时间）
2. 选择目标班级/学生
3. 设置评价指标关联
4. 发布任务
5. 学生上传作品后线上评价

#### 5.2.4 班级报告（P1）

**功能描述**：一键生成班级四维雷达图，快速定位班级整体优劣势

**功能特性**：

- 自动生成班级素养雷达图
- 六翼均分对比
- 班级优势/待提升维度分析
- 支持导出PDF报告

**报告内容**：

- 班级六翼得分分布图
- 与年级平均对比
- 学生个体排名（匿名）
- 改进建议

#### 5.2.5 评语库（P2）

**功能描述**：预设针对不同六翼表现的评语模板，提高效率

**功能特性**：

- 按六翼分类的评语模板（每翼20条）
- 智能推荐评语（基于评分自动匹配）
- 自定义评语收藏
- 常用语快捷输入

#### 5.2.6 数据看板（P2）

**功能描述**：查看班级六翼均分趋势图，追踪阶段性成长

**功能特性**：

- 时间维度筛选（周/月/学期/学年）
- 六翼发展趋势曲线
- 与年级平均对比
- 异常数据预警（如某学生连续下降）

### 5.3 家长/学生端功能

#### 5.3.1 个人成长树/星球（P0）

**功能描述**：游戏化激励机制，每次获得评价成长树长出新叶或获得星星

**用户故事**：
> 作为小学生，我希望看到自己的努力让成长树长大，获得成就感。

**功能特性**：

- 3D/2D可视化成长树/星球
- 不同六翼对应不同颜色/形态
- 获得评价时的动画反馈
- 成就徽章系统
- 等级进阶（幼苗→小树→大树→参天大树）

**等级体系**：
| 等级 | 名称 | 分数要求 | 视觉表现 |
|-----|------|---------|---------|
| Lv.1 | 幼苗 | 0-100分 | 小芽苗 |
| Lv.2 | 小树 | 101-300分 | 小树苗 |
| Lv.3 | 大树 | 301-600分 | 茂盛大树 |
| Lv.4 | 参天大树 | 600+分 | 参天大树+光环 |

#### 5.3.2 雷达图报告（P0）

**功能描述**：随时查看个人当学期、学年、整个小学阶段的动态雷达图

**功能特性**：

- 四维雷达图展示
- 时间轴切换（学期/学年/学段）
- 历史对比（与上学期对比）
- 雷达图动画演进
- 分享功能（生成精美卡片）

#### 5.3.3 电子档案袋（P1）

**功能描述**：上传作品，申请探究层加分

**支持作品类型**：

- 绘画作品（图片）
- 作文/文章（文字+图片）
- 演奏视频（视频）
- 环保时装秀照片（图片）
- 其他创意作品

**核心流程**：

1. 选择作品类型
2. 上传作品（支持多图/视频）
3. 填写作品说明
4. 选择关联六翼维度
5. 提交审核
6. 教师审核后获得加分

**审核时效**：教师需在48小时内完成审核

#### 5.3.4 实践活动打卡（P1）

**功能描述**：参与"校-家-社-村"活动，拍照打卡获取积分

**支持活动类型**：

- 社区环保活动
- 家家亲活动
- 志愿服务
- 传统文化体验
- 其他实践活动

**防作弊机制**：

- GPS定位验证
- 照片时间戳验证
- 人脸识别（可选）

#### 5.3.5 成长轨迹（P2）

**功能描述**：查看1-6年级六翼素养变化曲线

**功能特性**：

- 长期趋势曲线图
- 关键节点标记（获奖、重大突破）
- 学年对比
- 毕业总览报告

#### 5.3.6 家校建议（P2）

**功能描述**：接收教师推送的个性化成长建议和家庭实践任务

**功能特性**：

- 个性化建议推送（基于数据分析）
- 家庭实践任务
- 亲子活动推荐
- 反馈确认机制

## 6. 技术架构

### 6.1 系统架构图

| 协议  | HTTPS                   |
| 格式  | JSON                    |

### 7.2 核心接口

#### 评价相关

| 方法   | 路径                                      | 说明       |
|------|-----------------------------------------|----------|
| POST | `/v1/evaluations`                       | 创建评价记录   |
| GET  | `/v1/evaluations/{id}`                  | 获取评价详情   |
| GET  | `/v1/evaluations/student/{studentId}`   | 获取学生评价列表 |
| POST | `/v1/evaluations/batch`                 | 批量创建评价   |
| GET  | `/v1/evaluations/class/{classId}/stats` | 获取班级评价统计 |

**创建评价请求示例**：

```json
{
  "studentId": "stu_123456",
  "sceneId": "scene_art_festival_2026",
  "sceneType": "ACTIVITY",
  "wingId": "wing_art",
  "score": 85,
  "level": 2,
  "comment": "在艺术节表演中表现突出，舞蹈动作优美",
  "images": ["https://oss.example.com/img1.jpg"],
  "evaluatorId": "teacher_789"
}
```

#### 学生成长相关

| 方法  | 路径                                | 说明      |
|-----|-----------------------------------|---------|
| GET | `/v1/growth/{studentId}/tree`     | 获取成长树数据 |
| GET | `/v1/growth/{studentId}/radar`    | 获取雷达图数据 |
| GET | `/v1/growth/{studentId}/timeline` | 获取成长时间线 |
| GET | `/v1/growth/{studentId}/report`   | 生成成长报告  |

#### 作品相关

| 方法   | 路径                              | 说明       |
|------|---------------------------------|----------|
| POST | `/v1/works`                     | 上传作品     |
| GET  | `/v1/works/{id}`                | 获取作品详情   |
| GET  | `/v1/works/student/{studentId}` | 获取学生作品列表 |
| PUT  | `/v1/works/{id}/review`         | 审核作品     |

#### 管理相关

| 方法   | 路径                       | 说明       |
|------|--------------------------|----------|
| GET  | `/v1/admin/systems`      | 获取评价体系列表 |
| PUT  | `/v1/admin/systems/{id}` | 更新评价体系   |
| GET  | `/v1/admin/scenes`       | 获取评价场景列表 |
| POST | `/v1/admin/scenes`       | 创建评价场景   |
| GET  | `/v1/admin/export`       | 导出数据     |

### 8.2 关键数据实体

#### 用户实体

```kotlin
data class User(
    val userId: String,           // 用户ID
    val role: UserRole,           // TEACHER | STUDENT | PARENT | ADMIN
    val name: String,             // 姓名
    val avatar: String?,          // 头像
    val schoolId: String,         // 学校ID
    val classId: String?,         // 班级ID
    val grade: Int?,              // 年级
    val status: UserStatus,       // 状态
    val createdAt: Long           // 创建时间
)

enum class UserRole { TEACHER, STUDENT, PARENT, ADMIN }
enum class UserStatus { ACTIVE, INACTIVE }
```

#### 评价体系实体

```kotlin
data class EvaluationSystem(
    val systemId: String,
    val schoolId: String,
    val name: String,
    val sixWings: List<Wing>,              // 六翼配置
    val fourDimensions: List<String>,      // 四维维度
    val threeLevels: List<String>,         // 三阶等级
    val isDefault: Boolean                 // 是否默认
)

data class Wing(
    val wingId: String,
    val name: String,               // 六翼名称
    val icon: String,               // 图标
    val color: String,              // 主题色彩
    val description: String,        // 描述
    val weight: Double              // 权重
)
```

#### 评价记录实体

```kotlin
data class EvaluationRecord(
    val recordId: String,
    val studentId: String,          // 学生ID
    val teacherId: String,          // 教师ID
    val sceneId: String,            // 场景ID
    val sceneType: SceneType,       // DAILY | ACTIVITY | PROJECT | PRACTICE
    val wingId: String,             // 六翼ID
    val score: Int,                 // 分数
    val level: Int,                 // 1 | 2 | 3
    val comment: String?,           // 评语
    val images: List<String>,       // 图片
    val createdAt: Long,            // 创建时间
    val syncStatus: SyncStatus      // 同步状态
)

enum class SceneType { DAILY, ACTIVITY, PROJECT, PRACTICE }
enum class SyncStatus { PENDING, SYNCED, FAILED }
```

#### 作品实体

```kotlin
data class StudentWork(
    val workId: String,
    val studentId: String,          // 学生ID
    val title: String,              // 标题

    val files: List<String>,        // 文件
    val wingId: String,             // 关联六翼

    val score: Int?,                // 得分
    val feedback: String?,          // 反馈
    val reviewerId: String?,        // 审核人
    val createdAt: Long             // 时间
)

enum class WorkType { IMAGE, VIDEO, ARTICLE }
enum class WorkStatus { PENDING, APPROVED, REJECTED }
```

- 评价数据本地队列，联网后自动同步
- 图片压缩后暂存，联网后上传
- 离线状态下可查看历史数据
- 智能同步策略，优先同步重要数据

- 学生隐私数据加密存储（AES-256）
- 敏感操作权限控制
- 数据传输使用HTTPS/TLS 1.3
- 定期数据备份机制（每日增量，每周全量）
- 操作日志审计

### 9.4 兼容性

- Android 8.0 (API 26) 及以上
- 适配手机、平板多种屏幕尺寸
- 支持暗黑模式
- 无障碍访问支持
